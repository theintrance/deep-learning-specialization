### 하이퍼 파라미터

- 지난 주 NN 을 구성하면서 설정할 수 있는 여러가지 값들, 하이퍼 파라미터에 대해 배움
- 여러 튜닝을 진행하면서 제일 최적의 값을 찾아야 함

### 테스트 세트

- 훈련 세트 전부를 훈련에 사용할 수 있겠지만 일부는 모델에 대한 평가에 사용할 테스트 세트로 사용할 수 있음

| Training Set | Test Set |
| --- | --- |
| 99% | 1% |
- 또는 일부를 Training 과 Test 모두 사용하기도 함 → Dev set

| Training Set | Dev Set | Test Set |
| --- | --- | --- |
| 98% | 1% | 1% |
- 예전에는 60/20/20 처럼 Dev, Test Set의 비중이 엄청나게 높았으나 요즘은 99.5/0.25/0.25 처럼 작은 값이 선호됨

### 훈련 세트와 테스트 세트 나누기

- 고양이 분류기를 나눌 때 훈련 세트를 웹 페이지에서 불러올 수 있을 것임
- 그렇게 모델을 구성한 후 직접 제작한 테스트 세트로 모델을 평가하면 잘 작동하지 않을 수 있음
- 그러므로, Dev Set 과 Test Set 의 distribution 을 통일 시킬 것

### 테스트 세트는 없어도 괜찮을 수 있다 (Dev set only)

- 일반적으로 테스트 세트는 모델이 실전 환경에서 얼마나 잘 작동하는지를 평가하기 위해 사용됨.
- 바이어스 추정이 필요하지 않은 경우 테스트 세트가 없어도 됨.
- 하지만 모델을 반복적으로 개발하고 튜닝하는 과정에서는 Dev Set 만으로도 충분할 수 있음.
    - Dev Set: 모델 개발 및 하이퍼 파라미터 튜닝에 사용.
    - Test Set: 최종 평가를 위해 남겨두는 것이 일반적이지만, 초기 개발 단계에서는 생략 가능.
- 테스트 세트를 사용하지 않는 경우:
    - 모델이 연구 단계에 있을 때: 아직 실전 적용 전, 실험적 모델인 경우.
    - 테스트 세트 확보가 어려운 경우: 데이터가 제한적일 때 Dev Set로만 평가.
    - 빠른 반복과 개발을 위해: 테스트 세트 없이 빠르게 모델을 개선하고 검증하고자 할 때.
    - **장점:**
        - 개발 속도 향상: Dev Set만을 사용하면 평가 주기가 짧아져 모델 개발 속도가 빨라질 수 있음.
        - 자원 절약: 데이터 분할과 관리가 간편해지고, 추가적인 테스트 데이터가 필요 없음.
    - **단점:**
        - 과적합 가능성: Dev Set만을 사용하면 그 데이터에 과적합될 위험이 있음.
        - 일반화 성능 확인 어려움: 실제 환경에서의 성능을 평가할 독립적인 데이터가 부족함.
