---
title: 잘못 레이블이 지정된 데이터 정리
---
![](/assets/02cc6d09-0afe-4f55-a9e2-78bb93e27cda.png)

- 고양이 분류기의 데이터 셋에 잘못된 레이블이 들어간 예제가 있을 수 있음
- 딥러닝은 의외로 랜덤하게 잘못된 레이블에 대해서 잘 처리함
    - 오류가 합리적으로 랜덤하다면, 그대로 두어도 됨
    - 단 딥러닝은 시스템적인 오류에 대해서는 잘 처리하지 못함
        - 레이블 작업자가 흰 강아지에 대해서 계속 고양이라고 라벨링을 했으면 문제가 된다.

- 수동 오류 분석 테이블에 잘못 라벨링된 열을 추가하여 오류 비율이 크다면, 재 라벨링이 필요할 수 있음

**확인 순서 예시 1):**

1. dev set error 를 확인한다. → 10%
2. 잘못 라벨링 된 비율을 확인한다. → 0.6%
3. 다른 케이스를 확인한다 → 9.4%
4. 잘못 라벨링된 예시라 실패한 항목들의 비율이 적으므로 재 라벨링은 권장되지 않는다.

**확인 순서 예시 2):**

1. dev set error 를 확인한다. → 2%
2. 잘못 라벨링 된 비율을 확인한다. → 0.6%
3. 다른 케이스를 확인한다 → 1.4%
4. 잘못 라벨링 된 비율이 2% 중 0.6% 이므로 오류 중에서는 30% 에 육박한다. 수동 재 라벨링이 권장된다.

- 위와 같은 과정을 개발/테스트 세트 둘 다 에서 수행하는 것이 좋다.
    - 서로 같은 분포를 사용해야하기 때문
- 머신러닝 엔지니어들은 수동 오류 분석을 하기 싫어하는 경향이 있다
    - 응 선생님도 직접한다
    - 게으르게 일하지마라
